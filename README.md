# VoiceOn

**「英数」キーを押している間だけ音声入力**して、離すと**カーソル位置に文字を打ち込む** macOS 常駐アプリ（メニューバー）。
個人の趣味で作ったツールです。無保証ですが、よければ使ってみてください。

- 🎙️ Apple の Speech フレームワーク（オンデバイス）を使用 → **完全無料・オフライン・通信なし**
- ⌨️ 認識結果を「キー入力」として送るので **ほぼ全アプリで動く**（ターミナル / VS Code / ブラウザ等）
- ⚡ **キーを離した瞬間に確定テキストを一括挿入**（入力中のガタつきが無い）。**句読点も自動付与**
- 🔴 録音中はマウスカーソルに**音量連動インジケータ**（声が大きいほど赤く・大きく）
- 🖱️ トリガーは2種類：**「英数」長押し**（日本語配列）／**⌘ダブルタップ＆ホールド**（US/JIS両対応）

---

## 動作環境

- macOS 13 以降（Apple Silicon / Intel）
- Xcode Command Line Tools（Swift コンパイラ）

---

## クイックスタート（ゼロから）

### 1. 開発ツールを用意（初回のみ）
```sh
xcode-select --install      # 未導入なら。Swift コンパイラが入ります
```

### 2. 取得してビルド
```sh
git clone <このリポジトリのURL> voice_on
cd voice_on
./build.sh                  # VoiceOn.app が生成されます
```

### 3. 起動
```sh
open VoiceOn.app            # メニューバーに 🎙️ が出ます
```

### 4. 権限を許可（初回のみ）
起動時に2つのダイアログが出ます。どちらも許可してください。
- **音声認識**
- **マイク**

さらに **アクセシビリティ** の許可が必要です：
1. `システム設定 > プライバシーとセキュリティ > アクセシビリティ` を開く
2. `VoiceOn.app` を **「＋」で追加**してトグルを **ON**
3. アプリを再起動（`open VoiceOn.app`）

> 既に同名の項目があって効かない場合は、一度 **削除（−）してから追加し直す**と確実です。

### 5. 使ってみる
テキスト欄にカーソルを置いて、下記のどちらかで話します。

- **「英数」キーを長押し**（200ms〜）→ 話す → **離す**
- **⌘ を1回タップ → すぐもう一度押して長押し** → 話す → **離す**

短く「英数」を押した場合は通常どおり IME の英字切替として動きます。

## 設定（`Sources/VoiceOn/main.swift` の `Config`）

変更したら `./build.sh` で反映されます。

| 項目 | 既定 | 説明 |
|---|---|---|
| `triggerKeycode` | 102（英数） | トリガーキー |
| `holdThresholdMs` | 200 | 長押し判定（ms） |
| `localeIdentifier` | `ja-JP` | 認識言語 |
| `cmdGestureEnabled` | true | ⌘ダブルタップ＆ホールドの有効/無効 |
| `indicatorPosition` | `.bottomLeft` | 録音インジケータの位置（topCenter / bottomCenter / bottomLeft / topRight / mouse） |
| `cursorRingColor` / `cursorCoreColor` | 白 / 赤 | リング色 / 中心ドット色 |
| `cursorBaseDiameter` | 30 | リング基準サイズ（音量で拡大） |
| `cursorAnimationFile` | nil | カーソルに重ねる GIF 等の絶対パス（指定するとリングの代わりに表示） |
| `levelGain` / `levelCurve` | 28 / 0.6 | 音量感度（大きいほど／小さいほど敏感） |
| `segmentMinChars` | 3 | 区切り判定のしきい値（二重登録・消失の調整） |

## 署名・配布について

- このリポジトリは **ソースからビルドして使う** 形を想定しています。
- `build.sh` は、ローカルに自己署名の証明書（`VoiceOn Local Signing`）があればそれで署名し、無ければ **ad-hoc 署名にフォールバック**します。
- ad-hoc 署名のままだと **再ビルドのたびにアクセシビリティ権限の付与し直しが必要**です。毎回の再付与を避けたい場合は、自己署名のコード署名証明書を作成して使ってください（キーチェーンアクセスの「証明書アシスタント」で作成、種別＝コード署名）。
- ビルド済みアプリの配布は Gatekeeper に阻まれるため行っていません。きちんと配布する場合は Apple Developer Program による署名＋notarize が必要です。

---

## 既知の制限

- Apple のストリーミング認識は、無音などで途中経過がリセットされる挙動があり、ごく稀に文字の重複/欠けが起きることがあります（`segmentMinChars` で軽減）。
- 「英数」キーは日本語配列のキーで、US配列には物理キーがありません。US配列では **⌘ダブルタップ＆ホールド** を使ってください。
- セグメントの境目では句読点が付きにくいことがあります。
